# PoseCare Demo｜专属体态人格

拍一张全身体态照，生成你的专属体态人格，并获得拍照姿势、穿搭风格、放松运动和状态记录建议。

PoseCare Demo 目前不是传统的姿势矫正工具，也不是单纯的穿搭推荐工具。它把浏览器端体态识别结果转译成更温和、更审美化的「身体气质」表达，让用户用低焦虑的方式理解自己的体态特点，并围绕体态人格获得可保存、可分享的生活方式内容。

## 项目定位

本项目面向关注自我表达、拍照出片、穿搭风格和轻量身体照顾的年轻用户，尤其适合：

- 想知道自己适合什么拍照姿势、穿搭轮廓和放松方式的用户。
- 不喜欢传统体态检测里“缺陷”“风险”“矫正”话术的用户。
- 希望用状态卡、小手账等形式记录身体状态和生活情绪的用户。
- 对相似体态灵感、同风格人群内容和社区连接有兴趣的用户。

产品核心主线是：

```text
上传 / 拍摄全身体态照片
→ 浏览器端 AI 姿态识别
→ 生成专属体态人格
→ 输出体态人格小手账
→ 生成今日状态卡与本周体态情绪曲线
→ 展示相似体态社区预览
```

同时，项目保留了早期的「姿势问诊镜」能力，作为第二个模块接入当前前端：用户可以基于同一张照片提问“我这个坐姿哪里不对？”，系统会输出颈部、肩线、腰背的风险解释和调整建议。

## 解决的痛点

传统姿态检测产品往往偏医学、偏纠错，容易让用户感到被评价；而普通穿搭或拍照建议又过于通用，无法结合用户真实的身体线条、站姿重心和肩颈状态。

PoseCare Demo 解决的问题是：如何把体态识别从“指出问题”改成“发现适合你的表达方式”。项目不强调身材缺陷、不输出医学诊断，而是把肩颈舒展度、重心稳定感、线条延展感、松弛感等信息转化为人格、风格和日常建议。

用户获得的价值包括：

- 生成一个稳定的「专属体态人格」，把身体气质变成可识别、可收藏的身份资产。
- 获得围绕人格生成的拍照姿势、穿搭指南和放松运动建议。
- 通过今日状态卡记录身体情绪，并支持一键分享。
- 通过本周体态情绪变化曲线查看近期状态变化。
- 通过相似体态社区预览看到未来可扩展的同人格灵感内容。

## 当前功能

- 全屏视觉首页：使用角色轮播作为产品封面，用户点击 `DISCOVER IT` 进入内容页。
- 图片上传：用户可以上传一张全身体态照片并自动触发分析。
- 现场拍照：移动端可通过系统相机拍摄一张照片；桌面端会走系统文件选择能力。
- 浏览器端姿态识别：使用 MediaPipe Pose Landmarker 读取人体关键点。
- 专属体态人格：生成体态人格名称、Posture ID、气质描述、关键词和核心优势。
- 体态气质档案：展示肩颈舒展、重心平衡、线条延展、松弛感、气质外放、站姿稳定等指标。
- 体态人格小手账：提供拍照姿势推荐、穿搭指南、放松运动指南。
- 今日状态卡：生成今日体态情绪、状态值、关键词和治愈文案，并支持一键分享。
- 本周状态卡：基于今日结果生成一周体态情绪变化曲线和下周建议。
- 相似体态社区预览：展示同人格小组、相似度和灵感内容入口。
- 姿势问诊镜 Beta：基于同一张照片分析坐姿风险，并回答用户关于姿势问题的提问。

## 体验流程

### 1. 进入产品

打开页面后首先看到全屏视觉封面。点击右下角 `DISCOVER IT` 进入产品内容页。

内容页顶部有两个模块入口：

- `专属体态人格`：主产品模块，用于生成体态人格、小手账、状态卡和社区预览。
- `姿势问诊镜 Beta`：辅助工具模块，用于分析坐姿风险并回答姿势问题。

### 2. 体验专属体态人格

1. 进入内容页后默认停留在 `专属体态人格` 模块。
2. 点击 `上传全身照`，选择一张清晰、单人、尽量露出头部、肩膀、骨盆和腿部的照片。
3. 移动端也可以点击 `现场拍一张照`，调用系统相机拍摄一张照片。
4. 上传后系统会自动进行浏览器端姿态识别，并生成专属体态人格。
5. 结果页会展示人格名称、Posture ID、体态关键词、气质描述和六项体态指标。
6. 继续向下可以查看 `体态人格小手账`，包括拍照姿势推荐、穿搭指南和放松运动指南。
7. 页面下方会生成 `今日体态状态卡`、`本周体态情绪变化曲线` 和 `相似体态社区预览`。
8. 在今日状态卡中点击 `一键分享`，支持系统分享；不支持系统分享的浏览器会尝试复制分享文案。

### 3. 体验姿势问诊镜 Beta

1. 点击模块切换中的 `姿势问诊镜 Beta`。
2. 使用同一张已上传照片，或重新点击 `上传全身照 / 现场拍一张照`。
3. 在输入框中填写问题，例如：`我这个坐姿哪里不对？`、`怎么调整肩颈？`、`腰背会不会压力大？`
4. 点击 `开始问诊`。
5. 系统会先用本地姿势规则计算颈部、肩线、腰背风险，并展示风险分数、关键发现和调整建议。
6. 如果后端 AI 接口可用，系统会调用 `/api/posture-explain` 生成更自然的中文解释。
7. 如果 AI 接口不可用，前端仍会展示本地规则生成的兜底建议，保证演示流程不中断。

### 4. 推荐演示图片

为了保证识别稳定，建议使用：

- 单人照片。
- 光线清晰、身体无遮挡。
- 头部、肩膀、骨盆和腿部尽量完整入镜。
- 体态人格模块优先使用全身站姿照。
- 姿势问诊镜模块优先使用清晰坐姿照。

## 实现方案

项目采用 React + TypeScript + Vite 构建前端界面，核心识别逻辑尽量放在浏览器端完成，降低隐私风险并提升演示稳定性。

关键实现模块：

- `src/App.tsx`：主交互流程、首页轮播、图片上传 / 拍照入口、结果页渲染。
- `src/pose/poseLandmarker.ts`：加载并调用 MediaPipe Pose Landmarker，提取人体姿态关键点。
- `src/pose/bodyPersona.ts`：将关键点转化为体态指标，并生成体态人格、小手账、状态卡和社区预览数据。
- `src/pose/postureRules.ts`：将关键点转化为坐姿风险指标，并生成姿势问诊镜的本地兜底建议。
- `src/styles.css`：产品视觉样式，包括首页封面、内容页卡片、渐变模块和响应式布局。
- `api/` 与 `server/`：保留服务端 API 能力，可用于后续接入大模型解释、健康检查和 Vercel Serverless Functions。

当前版本的体态人格生成主要基于浏览器端姿态关键点和本地规则完成。原始照片不会被发送给大模型；如果后续接入 AI 文案生成，也应优先发送结构化指标，而不是直接上传用户原图。

## 核心创新点

1. **从姿态检测到体态人格**

   项目不把用户身体描述成“问题清单”，而是生成一个稳定的体态人格，例如晨雾天鹅型、暖阳小鹿型、山茶舒展型等。人格成为后续内容推荐、状态记录和社区连接的基础。

2. **把检测报告改造成生活方式小手账**

   结果页不是冷冰冰的检测分数，而是围绕“如何穿、如何拍、如何照顾身体”组织内容，让体态识别直接转化为用户可以使用的日常建议。

3. **低焦虑、非审判式表达**

   产品避免“驼背严重”“高低肩明显”“体态差”等负面措辞，改用“肩颈有些疲惫”“线条还有打开空间”“今天的身体需要被照顾”等表达，降低身材焦虑。

4. **状态卡与周期曲线**

   专属体态人格是稳定身份，今日状态卡和本周曲线则表达动态变化。这个结构让产品既有首次生成的惊喜，也有持续使用的记录价值。

5. **隐私优先的识别链路**

   姿态关键点提取运行在浏览器端，当前核心分析不依赖上传原始照片到第三方模型。产品也不展示体重、三围等敏感数据，不做医学诊断。

## 隐私与边界

- 本项目用于生活方式建议和产品概念演示，不替代医生诊断。
- 不输出疾病判断、医学结论或身体缺陷评价。
- 不展示体重、三围等敏感身体数据。
- 当前核心分析基于浏览器端姿态关键点完成，原始照片不发送给大模型。
- 社区能力目前为前端预览概念，不会公开用户原始照片。

## 本地运行

```bash
npm install
npm run dev
```

默认会同时启动 Vite 前端和本地 Node 服务。

如需配置后端 AI 文案接口，可参考 `.env.example` 创建 `.env`：

```bash
AI_PROVIDER=openai
AI_API_KEY=your_openai_api_key_here
AI_BASE_URL=https://api.openai.com/v1
AI_MODEL=gpt-4o-mini
```

> 模块 1 的「看图生成体态人格」需要支持视觉输入的模型。当前推荐使用 `gpt-4o-mini`，它会接收压缩后的图片和浏览器端姿态指标，动态生成体态人格、小手账和状态卡。模块 2 的姿势问诊镜也复用同一套 AI 配置生成中文解释。

## 构建

```bash
npm run build
```

## 部署

项目已适配 Vercel：

```bash
Build Command: npm run build
Output Directory: dist
Install Command: npm install
```

Vercel 会将 Vite 前端构建到 `dist/`，并将 `api/` 下的接口作为 Serverless Functions 部署。部署后可访问 `/api/health` 检查服务端环境。

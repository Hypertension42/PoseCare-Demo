# PoseCare MVP 执行计划

## 当前结论

为了比赛快速搓 Demo，先做：

> 移动端 Web/H5 + 坐姿检测主流程 + 姿势体检卡

不要先做完整医疗系统，也不要先做纯聊天 Agent。

## 第一版必须做

### 1. 首页即相机页

功能：

- 模拟手机相机取景框。
- 选择检测场景，默认“久坐姿势”。
- 提供“上传图片”和“使用样例”两个入口。
- 输入框预填：“我这个坐姿哪里不对？”

第一版可以先不做真摄像头，先用样例图把流程跑通。

### 2. 分析结果页

功能：

- 显示用户图片或样例图。
- 叠加人体骨架线。
- 标出颈部、肩膀、腰背三个风险点。
- 展示风险等级：中等风险。
- 展示核心问题：
  - 头前伸明显。
  - 肩膀轻微前扣。
  - 躯干前倾，腰背压力增加。

### 3. AI 建议区

功能：

- 回答用户问题。
- 给出简短医学解释。
- 给出 3 条纠正建议。
- 给出医疗边界提示。

第一版用本地模板生成，不依赖真实大模型接口。

### 4. 姿势体检卡

功能：

- 显示项目名 PoseCare。
- 显示风险等级。
- 显示 3 个风险点。
- 显示 3 条建议。
- 适合截图分享。

第一版可先做页面内卡片，不急着做下载图片。

## 第二版再做

- 接入真实摄像头。
- 接入 MediaPipe Pose Landmarker 或 MoveNet。
- 用真实关键点替代手写骨架。
- 接入浏览器语音识别。
- 接入大模型，把模板建议升级成 Agent 回答。

## 技术实现优先级

1. 先搭 Vite + React + TypeScript。
2. 先做 UI 和完整页面流。
3. 用样例数据画骨架和风险点。
4. 再接摄像头/上传。
5. 再接姿态识别模型。
6. 最后接 Agent。

## 推荐页面状态

```text
Capture
  -> Analyzing
  -> Result
  -> ReportCard
```

四个状态在一个单页应用里完成即可，不需要路由。

## 答辩演示脚本

1. “我们做的是医学健康方向的视觉搜索，不是问 AI 标准坐姿是什么，而是让 AI 看懂我现在的姿势。”
2. 打开网页，展示相机/上传页。
3. 选择“久坐姿势”，点击“使用样例”或上传现场照片。
4. 输入或说出：“我这个坐姿哪里不对？”
5. 展示骨架标注和风险点。
6. 展示 AI 建议。
7. 生成姿势体检卡。
8. 强调边界：“这是姿势风险筛查，不替代医生诊断。”

## 不做事项

第一版不要做：

- 登录注册。
- 病历管理。
- 医院预约。
- 复杂多用户后台。
- 真实医学诊断。
- 大而全的健康 App。

这些会稀释赛道二核心体验。
